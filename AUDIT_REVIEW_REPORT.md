### 修复方案说明

**C-01 页面组件不渲染** — 已修复

根因：Dash 框架要求回调引用的组件 ID 必须在初始 layout 中可发现。原架构使用 `navigate` 回调动态返回页面内容，导致交互组件 ID 不在初始 layout 中，被 Dash 客户端静默丢弃。

修复方案：
1. 所有 8 个页面在初始 layout 中预声明，通过 `display: none/block` 切换可见性
2. 修复 `_section()` 函数的 `children` 列表嵌套 bug（`extend()` 替代嵌套 append）

**导航二次点击失效** — 已修复

根因：Dash 回调系统在更新按钮 `style` 属性时，会重新渲染组件，导致 `n_clicks` / `n_clicks_timestamp` 状态追踪异常。尝试了多种 Dash 回调模式（html.Button + n_clicks, dcc.Link + dcc.Location, n_clicks_timestamp）均无法可靠解决。

最终修复方案：**纯 JavaScript 客户端导航**
- 使用 `html.Button` 作为导航按钮
- 注入客户端 JavaScript 监听按钮点击事件
- JavaScript 直接操作 DOM 切换页面可见性和导航高亮
- 使用 `history.pushState()` 更新 URL
- 完全绕过 Dash 回调系统，避免状态追踪问题
- 支持浏览器前进/后退按钮（`popstate` 事件监听）

**C-02 API 依赖注入** — 已修复

在 `app.py` 中添加了全局依赖注入，将 `backtest_engine`、`strategy_registry`、`market_overview` 实例传递给 API 模块和 Dashboard。
